# Docker
## Create and enter docker
e.g.
```sh
docker run --name RzH -it -d --net=host --shm-size=500g \
    --privileged=true \
    -w /home \
    --device=/dev/davinci_manager \
    --device=/dev/hisi_hdc \
    --device=/dev/devmm_svm \
    --entrypoint=bash \
    -v /usr/local/Ascend/driver:/usr/local/Ascend/driver \
    -v /usr/local/dcmi:/usr/local/dcmi \
    -v /usr/local/bin/npu-smi:/usr/local/bin/npu-smi \
    -v /usr/local/sbin:/usr/local/sbin \
    -v /home:/home \
    -v /tmp:/tmp \
    -v /data:/data \
    -v /data3:/data3 \
    -v /data1:/data1 \
    -v /core:/core \
    -v /usr/share/zoneinfo/Asia/Shanghai:/etc/localtime \
    -e http_proxy=$http_proxy \
    -e https_proxy=$https_proxy \
    ${REPOSITORY}:${TAG}

docker exec -itu root ${NAME} bash
```

说明：
1. `-it`:
    - `-i`: 以交互模式运行容器，保持便准输入(STDIN)打开。
    - `-t`: 为容器分配一个伪终端。
2. `-d`: 使容器在后台运行(脱离终端)。
3. `--net-host`: 设置容器使用主机的网络模式。容器将共享主机的网络接口，这意味着容器内的应用程序将能够访问主机的所有网络服务。
4. `--shm-size=500g`: 设置容器的共享内存大小为 500GB。如果容器需要大量的共享内存（如一些机器学习应用），就需要指定这个值。
5. `--privileged=true`: 赋予容器更多的特权权限，这通常用于需要访问主机设备或运行特殊系统操作的容器。
6. `-w /home`: 设置容器内的工作目录为 /home，即容器启动后，默认进入的目录。
7. `--device`: 允许容器访问主机的特定设备。容器可以直接与这些设备进行交互。通常用于需要硬件加速（如 NPU）的应用。
    - `--device=/dev/davinci_manager`, `--device=/dev/hisi_hdc`, `--device=/dev/devmm_svm`为常见ascend npu设置
8. `--entrypoint=bash`: 设置容器启动时的入口点为 bash。这意味着容器启动时，会直接进入 bash shell，而不是默认的应用程序。
9. `-v`: 用于挂载主机的目录到容器内
    - 它们把主机的文件系统目录（如 `/usr/local/Ascend/driver`）挂载到容器内的相应目录，使得容器能够访问主机的文件。
10. `-e`: 配置环境变量
11. 对于运行命令`docker exec -itu root ${NAME} bash`
    - `-it`:
        `-i`：保持标准输入打开。
        `-t`：分配一个伪终端。
    - `-u root`: 以 root 用户身份执行命令。
    - `bash`: 在容器内运行 bash shell。


2. rebase到最早的commit

```sh
git rebase -i e7e1806ab909d197a7f3c4e9ffb4d9a5b382ed19
```

然后编辑信息将第一个保留为pick，其余的改为s(squash)
保存退出后进入commit信息编辑，据需要保留

3. git push提交

- 提交到自己的仓库

```sh
git push origin master --force-with-lease
```

- 提交到pr

```sh
git push --force-with-lease
```
说明：
最终效果为e7e1806即modify rank...这一commit下包含了后续所有的squash提交。
